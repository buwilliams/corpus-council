# Task 00019: Unit tests — conversation and collection modules

## Role
tester

## Objective
Write `tests/unit/test_conversation.py` and `tests/unit/test_collection.py` with full unit test coverage of conversation mode orchestration and collection mode. All tests use real files in `tmp_path`, real `FileStore` I/O, real council persona files, and real templates. The only permitted mock is the HTTP transport layer for LLM calls. Must pass with `uv run pytest`.

## Context
Task 00011: `ConversationResult`, `run_conversation` in `src/corpus_council/core/conversation.py`.
Task 00012: `CollectionSession`, `start_collection`, `respond_collection`, `get_collection_status` in `src/corpus_council/core/collection.py`.
Task 00016: `tests/conftest.py` with `test_config`, `file_store`, `council_dir`, `templates_dir`, `plans_dir` fixtures.
Task 00018: `TestLLMClient` pattern — subclass `LLMClient` and override `call()` only; `render_template` remains real.

For collection validation, `TestLLMClient.call` for `"collection_validate"` must return a JSON string: `'{"valid": true, "extracted_value": "Alice", "reason": "ok"}'`. For `"collection_prompt"`, return a realistic prompt string.

**`tests/unit/test_conversation.py` must cover:**
- `test_conversation_run_creates_messages_jsonl`: single turn; assert `chat/messages.jsonl` exists with 1 record
- `test_conversation_run_updates_context_json`: single turn; assert `chat/context.json` exists with `turn_count == 1`
- `test_conversation_resume_loads_prior_context`: run two turns with same `user_id`; assert second result has `turn_count == 2`; assert `messages.jsonl` has 2 records
- `test_conversation_messages_jsonl_has_required_fields`: read `messages.jsonl` record; assert keys: `timestamp`, `user_message`, `deliberation_log`, `final_response`
- `test_conversation_returns_non_empty_response`: assert `result.response != ""`
- `test_conversation_different_users_isolated`: two different `user_id`s; assert their `messages.jsonl` files are at different paths and each contains only their own turn

**`tests/unit/test_collection.py` must cover:**
- `test_start_collection_creates_session_files`: call `start_collection`; assert `session.json`, `context.json`, and `collected.json` exist at correct paths
- `test_start_collection_returns_first_prompt`: assert `session.next_prompt` is a non-empty string
- `test_start_collection_with_invalid_plan_raises`: call with nonexistent `plan_id`; assert `FileNotFoundError`
- `test_respond_collection_advances_to_next_field`: plan with 2 required fields; first respond; assert `status == "active"` and `next_prompt != None`
- `test_respond_collection_completes_when_all_fields_collected`: respond to both fields; assert `status == "complete"` and `next_prompt == None`
- `test_respond_collection_saves_collected_json`: after completing, read `collected.json`; assert both field values present
- `test_respond_collection_appends_to_messages_jsonl`: respond twice; assert `messages.jsonl` has 2 records
- `test_get_collection_status_returns_correct_shape`: call `get_collection_status`; assert keys: `session_id`, `user_id`, `status`, `collected`, `created_at`
- `test_get_collection_status_raises_for_missing_session`: call with nonexistent session; assert `FileNotFoundError`

For collection tests, use a `CollectionTestLLM` that:
- Returns `'{"valid": true, "extracted_value": "TestValue", "reason": "ok"}'` for `collection_validate`
- Returns `"Please provide your {field_name}."` (with real template rendering) for `collection_prompt`
- Returns `"NOT_TRIGGERED"` for `escalation_check`
