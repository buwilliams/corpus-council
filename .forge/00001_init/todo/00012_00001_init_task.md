# Task 00012: Implement collection.py — structured collection mode orchestration

## Role
programmer

## Objective
Implement `src/corpus_council/core/collection.py` with functions for starting a collection session, processing user responses, and retrieving session state. The collection mode reads a structured interview plan from `plans/`, drives the user through required fields via council-validated prompts, accumulates collected values in `collected.json`, and marks the session `complete` when all required fields are gathered. Must pass `mypy --strict` and `ruff check`.

**Collection plan format** (`plans/{plan_id}.md`):
YAML front matter defines the fields to collect:
```yaml
---
title: "Plan title"
fields:
  - name: first_name
    description: "The user's first name"
    required: true
    validation_rule: "Must be a non-empty string"
  - name: age
    description: "The user's age in years"
    required: true
    validation_rule: "Must be a positive integer between 1 and 150"
---
Body prose providing context for the collection session.
```

**Session file schemas** (from `data-engineer.md`, Task 00003):
- `session.json`: `{"session_id": str, "user_id": str, "plan_id": str, "status": "active"|"complete", "created_at": ISO-8601, "completed_at": ISO-8601|null}`
- `collected.json`: `{"field_name": "collected_value", ...}`
- `context.json`: `{"session_id": str, "current_field": str, "fields_remaining": list[str], "last_updated": ISO-8601}`
- `messages.jsonl`: same turn schema as conversation

**Public functions to implement:**

```python
@dataclass
class CollectionSession:
    user_id: str
    session_id: str
    status: str  # "active" | "complete"
    collected: dict[str, Any]
    next_prompt: str | None  # None when complete

def start_collection(
    user_id: str, plan_id: str, session_id: str,
    config: AppConfig, store: FileStore, llm: LLMClient
) -> CollectionSession:
    """Create new session, generate first question, return session state."""

def respond_collection(
    user_id: str, session_id: str, message: str,
    config: AppConfig, store: FileStore, llm: LLMClient
) -> CollectionSession:
    """Process user response: validate+extract field value, advance to next field or complete."""

def get_collection_status(
    user_id: str, session_id: str, store: FileStore
) -> dict[str, Any]:
    """Return session.json + collected.json merged for API response."""
```

**`start_collection` pipeline:**
1. Load and validate plan using `validate_plan_id(plan_id, config)`.
2. Parse plan YAML front matter to get `fields` list.
3. Generate `session_id` (uuid4) if not provided (but caller from API always provides it).
4. Initialize session files via `store.write_json`.
5. Generate first prompt using `llm.call("collection_prompt", {...})`.
6. Return `CollectionSession`.

**`respond_collection` pipeline:**
1. Load `session.json` — raise `FileNotFoundError` if not found.
2. If `status == "complete"`, return current state immediately (idempotent).
3. Load `context.json` to get `current_field`.
4. Load the plan to get field definition.
5. Validate/extract value using `llm.call("collection_validate", {...})`. Parse JSON response to get `{"valid": bool, "extracted_value": str, "reason": str}`.
6. If valid: save to `collected.json`, advance `fields_remaining`.
7. If no more required fields: mark session `complete`, set `next_prompt = None`.
8. If more fields remain: generate next prompt with `llm.call("collection_prompt", {...})`.
9. Append turn to `messages.jsonl` via `store.append_jsonl`.
10. Return `CollectionSession`.

Use `from __future__ import annotations`. No inline prompt strings. All user data I/O through `store`.
