# Task 00000: Ensure .gitignore covers secrets and build artifacts

## Role
programmer

## Objective
Create `/home/buddy/projects/corpus-council/.gitignore` that covers all common secrets, credentials, and build artifacts. The file must include at minimum: `.env`, `.env.*`, `*.key`, `*.pem`, `*.p12`, `node_modules/`, `dist/`, `build/`, `target/`, `__pycache__/`, `*.pyc`, `.pytest_cache/`, `.mypy_cache/`, `.DS_Store`, `*.log`, `coverage/`, `.nyc_output/`. Also include Python-specific artifacts: `*.egg-info/`, `.venv/`, `uv.lock` is acceptable to commit but `.python-version` local overrides should be ignored.

## Context
The project root at `/home/buddy/projects/corpus-council/` currently has no `.gitignore`. This is a Python 3.12+ project managed by `uv` with FastAPI, pytest, mypy, and ruff. The project stores API keys exclusively in environment variables (never in files), but the `.gitignore` must ensure accidental key files cannot be committed. The forge directory is at `.forge/` and should not be in `.gitignore` since it is part of the project structure.

## Steps
1. Create `/home/buddy/projects/corpus-council/.gitignore` with all required entries listed in the Objective.
2. Add Python-specific entries: `*.egg-info/`, `.venv/`, `*.so`, `*.pyd`, `.python-version`.
3. Add IDE entries: `.idea/`, `.vscode/`, `*.swp`, `*.swo`.
4. Add `data/` to prevent accidental commit of generated user data, embeddings, and corpus chunks. Note: the `data/` directory is runtime-generated and must not be committed.
5. Stage only `.gitignore` and commit.
