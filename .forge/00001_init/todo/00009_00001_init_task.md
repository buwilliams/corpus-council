# Task 00009: Implement validation.py — input sanitization for user_id, session_id, plan_id

## Role
security-engineer

## Objective
Implement `src/corpus_council/core/validation.py` with a `validate_id(value: str, name: str) -> str` function and a `validate_path_containment(candidate: Path, parent: Path, label: str) -> Path` function. These must be called at every API endpoint and CLI command entry point before any value is passed to `FileStore` or filesystem operations. Must pass `mypy --strict` and `ruff check`.

## Context
Task 00001 created the scaffold. Task 00002 created `AppConfig`. Task 00003 created `FileStore`. The security requirements from `project.md` and `security-engineer.md` are:

- `user_id` and `session_id` must match `^[a-zA-Z0-9_-]{4,128}$` — reject anything outside this pattern
- `plan_id` must also match the same pattern (it maps to a file in `plans/`)
- Any caller-supplied filesystem `path` (e.g., from `POST /corpus/ingest`) must be resolved and confirmed to reside within the expected parent directory — preventing `../../etc/passwd` traversal attacks

`validate_id` implementation:
```python
import re
_SAFE_ID = re.compile(r'^[a-zA-Z0-9_-]{4,128}$')

def validate_id(value: str, name: str) -> str:
    if not _SAFE_ID.match(value):
        raise ValueError(f"{name} must be 4-128 alphanumeric/dash/underscore characters, got: {value!r}")
    return value
```

`validate_path_containment` implementation:
```python
def validate_path_containment(candidate: Path, parent: Path, label: str) -> Path:
    resolved = candidate.resolve()
    resolved_parent = parent.resolve()
    if not str(resolved).startswith(str(resolved_parent)):
        raise ValueError(f"{label} resolves outside expected directory: {resolved}")
    return resolved
```

Also add a `validate_plan_id(plan_id: str, config: AppConfig) -> Path` convenience function that:
1. Calls `validate_id(plan_id, "plan_id")`.
2. Constructs `config.plans_dir / f"{plan_id}.md"`.
3. Calls `validate_path_containment` on the result.
4. Checks the file exists; raises `FileNotFoundError(f"Plan not found: {plan_id}")` if not.
5. Returns the resolved path.

Use `from __future__ import annotations`.
