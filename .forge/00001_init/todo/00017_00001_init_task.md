# Task 00017: Unit tests — config, store, and corpus modules

## Role
tester

## Objective
Write `tests/unit/test_config.py`, `tests/unit/test_store.py`, and `tests/unit/test_corpus.py` with full unit test coverage of the config loader, FileStore, and corpus ingestion modules. All tests use real files in `tmp_path`. No mocks for file I/O, FileStore, or corpus loading. Coverage target: 80% on `src/corpus_council/core/`.

## Context
Task 00002: `AppConfig`, `load_config` in `src/corpus_council/core/config.py`.
Task 00003: `FileStore` in `src/corpus_council/core/store.py` — methods: `user_dir`, `append_jsonl`, `write_json`, `read_json`, `read_jsonl`, all `*_path` convenience methods.
Task 00004: `ingest_corpus`, `IngestResult` in `src/corpus_council/core/corpus.py`.
Task 00016: `tests/conftest.py` with `test_config`, `file_store`, `corpus_dir` fixtures.

Tester rules from `tester.md`:
- Never mock `FileStore`, corpus file loading, or file I/O
- Every test module covers: happy path, at least one edge case, at least one error/failure case
- Test names must describe what they assert: `test_load_config_returns_correct_paths`, not `test_load_config_1`
- Concurrent write test must actually run two threads simultaneously

**`tests/unit/test_config.py` must cover:**
- `test_load_config_returns_all_required_fields`: load real `config.yaml`; assert all `AppConfig` fields populated
- `test_load_config_resolves_paths_relative_to_config_file`: write a temp config.yaml with relative paths; assert `AppConfig.data_dir` resolves to absolute path relative to config file location
- `test_load_config_raises_file_not_found_for_missing_file`: assert `FileNotFoundError` raised for nonexistent path
- `test_load_config_raises_on_missing_required_key`: write a config.yaml missing `llm.provider`; assert `KeyError` or `ValueError` raised

**`tests/unit/test_store.py` must cover:**
- `test_user_dir_correct_sharding`: `user_id = "abc123ef"` → path ends with `ab/c1/abc123ef`
- `test_user_dir_raises_for_short_id`: `user_id = "ab"` → `ValueError`
- `test_append_jsonl_writes_record`: append a record, read back; assert equal
- `test_append_jsonl_multiple_records`: append 3 records; assert all 3 returned by `read_jsonl`
- `test_write_json_and_read_json_roundtrip`: write a dict, read it back; assert equal
- `test_read_json_returns_empty_dict_for_missing_file`: call `read_json` on nonexistent path; assert returns `{}`
- `test_read_jsonl_returns_empty_list_for_missing_file`: call `read_jsonl` on nonexistent path; assert returns `[]`
- `test_concurrent_appends_do_not_corrupt_data`: use `threading.Thread` to run 10 concurrent `append_jsonl` calls; assert final `read_jsonl` returns exactly 10 records (tests real fcntl locking)
- `test_write_json_is_atomic`: write a large dict; read it back immediately; assert no partial data

**`tests/unit/test_corpus.py` must cover:**
- `test_ingest_corpus_processes_md_and_txt_files`: create corpus with 2 `.md` and 1 `.txt`; assert `files_processed == 3`
- `test_ingest_corpus_ignores_non_corpus_files`: add a `.pdf` and `.py` file; assert `files_processed` does not count them
- `test_ingest_corpus_creates_chunk_json_files`: assert chunk JSON files exist at `data/chunks/{hash}/{index}.json`
- `test_ingest_corpus_chunks_respect_max_size`: create a 2000-char document with `chunk_max_size=200`; assert multiple chunks created, each `<= 200` chars
- `test_ingest_corpus_is_idempotent`: run twice on same corpus; assert second run `chunks_created == 0`
- `test_chunk_json_has_required_fields`: load a chunk file; assert keys: `chunk_id`, `source_file`, `source_hash`, `chunk_index`, `text`, `char_start`, `char_end`
