# Task 00008: Create prompt templates in templates/

## Role
programmer

## Objective
Create all LLM prompt template markdown files in `templates/`. These are the only place where LLM prompt text lives — none of it appears in Python source. Each template uses `{{ variable_name }}` Jinja2 syntax for interpolation. At minimum: `member_deliberation.md`, `escalation_check.md`, `escalation_resolution.md`, `final_synthesis.md`, `collection_prompt.md`, `collection_validate.md`.

## Context
Task 00001 created the `templates/` directory. Task 00007 created `LLMClient.render_template` which loads files from `config.templates_dir` and substitutes Jinja2-style variables.

The deliberation engine (to be implemented in Task 00010) needs these templates. The collection mode (Task 00012) needs collection templates. Template variables match what the calling Python code passes in the `context` dict.

**`member_deliberation.md`** — called once per council member during deliberation:
Variables: `{{ member_name }}`, `{{ persona }}`, `{{ primary_lens }}`, `{{ role_type }}`, `{{ user_message }}`, `{{ corpus_chunks }}` (formatted retrieved chunks), `{{ prior_responses }}` (formatted list of prior member outputs so far)

**`escalation_check.md`** — called to evaluate whether a member's escalation_rule was triggered:
Variables: `{{ escalation_rule }}`, `{{ member_response }}`
Expected output format: the LLM should respond with a single line `TRIGGERED` or `NOT_TRIGGERED` followed by optional explanation.

**`final_synthesis.md`** — called for position-1 member on normal path:
Variables: `{{ member_name }}`, `{{ persona }}`, `{{ primary_lens }}`, `{{ user_message }}`, `{{ corpus_chunks }}`, `{{ deliberation_log }}` (all prior member responses formatted)

**`escalation_resolution.md`** — called for position-1 member when escalation was triggered:
Variables: `{{ member_name }}`, `{{ persona }}`, `{{ primary_lens }}`, `{{ user_message }}`, `{{ corpus_chunks }}`, `{{ escalation_log }}` (which member triggered escalation and why), `{{ deliberation_log }}` (responses collected before escalation)

**`collection_prompt.md`** — called to generate the next interview question in collection mode:
Variables: `{{ field_name }}`, `{{ field_description }}`, `{{ collected_so_far }}`, `{{ conversation_history }}`

**`collection_validate.md`** — called to validate and extract a collected field value from user response:
Variables: `{{ field_name }}`, `{{ field_description }}`, `{{ user_response }}`, `{{ validation_rule }}`
Expected output: JSON object `{"valid": true/false, "extracted_value": "...", "reason": "..."}`

Templates should be professionally written but do not need to be LLM-perfect — they establish the structure that `LLMClient.render_template` will fill in at runtime.

## Steps
1. Create `templates/member_deliberation.md` with clear role instructions and all required variables.
2. Create `templates/escalation_check.md` instructing the LLM to output `TRIGGERED` or `NOT_TRIGGERED`.
3. Create `templates/final_synthesis.md` for position-1 final synthesis on normal path.
4. Create `templates/escalation_resolution.md` for position-1 resolution when escalation triggered.
5. Create `templates/collection_prompt.md` for generating collection interview questions.
6. Create `templates/collection_validate.md` for validating/extracting collected field values.
7. Verify each template file has the correct Jinja2 variable syntax (`{{ variable_name }}`).
