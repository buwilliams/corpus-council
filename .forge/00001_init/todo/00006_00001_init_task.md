# Task 00006: Implement council.py — persona loader and CouncilMember type

## Role
programmer

## Objective
Implement `src/corpus_council/core/council.py` with `load_council(config: AppConfig) -> list[CouncilMember]` that reads all `.md` files from `config.council_dir`, parses YAML front matter plus body prose, validates required fields, and returns a list of `CouncilMember` instances sorted by `position` ascending. The module must pass `mypy --strict` and `ruff check`.

## Context
Task 00001 created the scaffold. Task 00002 created `AppConfig`. The `council/` directory holds persona markdown files with YAML front matter matching this template:

```yaml
---
name: Role label (e.g. "Adversarial Critic")
persona: Who this agent is — their personality, domain, background, and worldview
primary_lens: The dimension they are responsible for judging
position: 1  # numeric rank — lower is more authoritative; position 1 has final say
role_type: proponent | critic | synthesizer | arbiter | devil's_advocate | domain_specialist
escalation_rule: The core rule that, if violated, halts deliberation
---

Body prose goes here. This is passed as additional context to the member's LLM prompt.
```

`CouncilMember` is a `dataclass`:
```python
@dataclass
class CouncilMember:
    name: str
    persona: str
    primary_lens: str
    position: int
    role_type: str
    escalation_rule: str
    body: str  # prose below the front matter
    source_file: str  # relative path to the .md file
```

Required fields: `name`, `persona`, `primary_lens`, `position`, `role_type`, `escalation_rule`. If any is missing from the YAML front matter, raise `ValueError(f"Council file {path} missing required field: {field}")`.

The `position` field must be an integer. If it cannot be cast to `int`, raise `ValueError`.

Use the `python-frontmatter` library (already in `pyproject.toml` from Task 00001) to parse front matter: `import frontmatter; post = frontmatter.load(str(path))`.

Files are opened in read mode (`"r"`) only — never write to `council/`.

Sort result by `position` ascending (position 1 first) before returning.

Use `from __future__ import annotations`.

## Steps
1. Create `src/corpus_council/core/council.py` with `from __future__ import annotations`.
2. Import `frontmatter`, `Path`, `dataclass`, `AppConfig` from `.config`.
3. Define `CouncilMember` dataclass with all fields listed in Context.
4. Implement `_parse_member(path: Path, config_dir: Path) -> CouncilMember` that loads a single file and validates required fields.
5. Implement `load_council(config: AppConfig) -> list[CouncilMember]` that:
   a. Globs `config.council_dir` for `*.md` files.
   b. Calls `_parse_member` for each.
   c. Sorts by `position` ascending.
   d. Returns the sorted list.
6. Run `uv run mypy src/corpus_council/core/council.py` and fix all errors.
7. Run `uv run ruff check src/corpus_council/core/council.py` and fix all errors.
