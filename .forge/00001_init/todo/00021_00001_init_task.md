# Task 00021: Integration tests — full conversation and collection flows

## Role
tester

## Objective
Write `tests/integration/test_full_conversation_flow.py` and `tests/integration/test_full_collection_flow.py` with end-to-end integration tests that exercise the complete conversation and collection pipelines against real test corpora, real council persona files, real ChromaDB instances in `tmp_path`, and real templates. The only permitted mock is the HTTP transport layer for LLM calls. Must pass with `uv run pytest`.

## Context
Task 00004: `ingest_corpus` in `src/corpus_council/core/corpus.py`.
Task 00005: `embed_corpus` in `src/corpus_council/core/embeddings.py`.
Task 00011: `run_conversation` in `src/corpus_council/core/conversation.py`.
Task 00012: `start_collection`, `respond_collection`, `get_collection_status` in `src/corpus_council/core/collection.py`.
Task 00016: `test_config`, `file_store`, `corpus_dir`, `council_dir`, `plans_dir`, `templates_dir` fixtures.
Task 00018: `TestLLMClient` pattern.

These tests exercise the complete pipeline:
- Corpus ingestion (`ingest_corpus`) with real files
- Embedding to ChromaDB (`embed_corpus`) with real sentence-transformers and real ChromaDB in `tmp_path`
- Retrieval (`retrieve_chunks`) returning real results
- Council loading (`load_council`) from real persona files
- Deliberation (`run_deliberation`) with `TestLLMClient`
- Persistence to real `messages.jsonl` and `context.json` via real `FileStore`

**`tests/integration/test_full_conversation_flow.py` must cover:**
- `test_two_turn_conversation_persists_correctly`:
  1. Ingest corpus, embed into ChromaDB.
  2. Run first conversation turn for `user_id = "user0001"`.
  3. Assert `messages.jsonl` has 1 record.
  4. Run second conversation turn for same `user_id`.
  5. Assert `messages.jsonl` has 2 records.
  6. Assert second result `turn_count == 2`.
- `test_conversation_resumes_context_from_disk`:
  1. Run a conversation turn, capture `turn_count`.
  2. Create a new `FileStore` instance (simulate process restart).
  3. Run another turn with the new store instance.
  4. Assert turn count incremented from the persisted context.
- `test_conversation_retrieves_relevant_corpus`:
  1. Create a corpus with specific content (e.g., a document about Python).
  2. Run a conversation with a question about Python.
  3. Assert the `deliberation_log` shows the council received non-empty `corpus_chunks` context.
  4. (Check by asserting `TestLLMClient.calls` included a `member_deliberation` call where `corpus_chunks` context is not the "no chunks available" placeholder.)

**`tests/integration/test_full_collection_flow.py` must cover:**
- `test_collection_completes_when_all_fields_provided`:
  1. Start a collection session for `user_id = "user0002"` with `plan_id = "signup"`.
  2. Respond with values for each required field.
  3. Assert final `status == "complete"`.
  4. Assert `collected.json` contains all required fields.
- `test_collection_session_persists_across_calls`:
  1. Start a session.
  2. Simulate process restart by creating a new `FileStore` instance.
  3. Call `respond_collection` with the new store.
  4. Assert session state is loaded from disk (not restarted).
- `test_collection_messages_jsonl_has_one_record_per_turn`:
  1. Plan with 2 required fields.
  2. Respond twice.
  3. Assert `messages.jsonl` has exactly 2 records.
- `test_get_collection_status_reflects_completed_session`:
  1. Complete a collection session.
  2. Call `get_collection_status`.
  3. Assert `status == "complete"` and `collected` has all field values.

The `TestLLMClient` for these tests should cycle through extracted values: for `collection_validate`, always return `'{"valid": true, "extracted_value": "TestValue123", "reason": "ok"}'`.
