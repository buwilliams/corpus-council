# Task 00007: Implement llm.py — LLM client with markdown template rendering

## Role
programmer

## Objective
Implement `src/corpus_council/core/llm.py` with an `LLMClient` class that loads LLM provider/model from `AppConfig`, reads API keys exclusively from environment variables, renders markdown templates from `config.templates_dir` before every call, and returns the LLM's text response. Must pass `mypy --strict` and `ruff check`. No inline prompt strings anywhere in this module.

## Context
Task 00001 created the scaffold and `templates/` directory. Task 00002 created `AppConfig`. Task 00006 created `CouncilMember`.

Template rendering:
- Templates are `.md` files in `config.templates_dir`
- Template variables are substituted using Python string `.format(**context)` or a Jinja2 environment with `autoescape=False` (user content is passed as variables, never concatenated into the template string itself)
- Preferred approach: use Python's `str.format_map()` with a `SafeDict` that leaves unknown keys as `{key}` rather than raising `KeyError`. This prevents partial-substitution errors.
- Alternatively, use Jinja2's `Environment(loader=FileSystemLoader(str(templates_dir)), autoescape=False)` — user-supplied values are passed as template variables, never as template strings.

`LLMClient` interface:
```python
class LLMClient:
    def __init__(self, config: AppConfig) -> None: ...

    def render_template(self, template_name: str, context: dict[str, Any]) -> str:
        """Load templates/{template_name} and substitute context variables."""
        ...

    def call(self, template_name: str, context: dict[str, Any]) -> str:
        """Render the template then call the configured LLM provider. Return text response."""
        ...
```

Provider implementations:
- `anthropic`: Import `anthropic` SDK. Read API key from `os.environ["ANTHROPIC_API_KEY"]` — raise `RuntimeError("ANTHROPIC_API_KEY environment variable is not set")` if absent. Use `anthropic.Anthropic(api_key=key).messages.create(model=config.llm_model, max_tokens=2048, messages=[{"role": "user", "content": rendered_prompt}])`. Return `response.content[0].text`.
- Other providers: raise `ValueError(f"Unknown LLM provider: {config.llm_provider}")`.

`render_template` must:
1. Construct path: `config.templates_dir / template_name` (template_name may include `.md` extension or not — normalize by adding `.md` if extension is absent).
2. Raise `FileNotFoundError` with a clear message if the template file does not exist.
3. Load the file content as UTF-8 text.
4. Substitute `context` variables. Use Jinja2 `Environment` or `str.format_map` with a safe fallback. User-supplied content is a variable, never part of the template string.
5. Return the rendered string.

`call` renders the template then dispatches to the configured provider.

Use `from __future__ import annotations`. Store no API keys in instance variables beyond what's needed for the immediate call. No LLM prompt text as Python string literals.
