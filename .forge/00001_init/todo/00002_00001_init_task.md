# Task 00002: Implement config.py — config.yaml loader with typed dataclass

## Role
programmer

## Objective
Implement `src/corpus_council/core/config.py` with a `load_config(path: str | Path) -> AppConfig` function that reads `config.yaml` via PyYAML and returns a fully typed `AppConfig` dataclass. All path fields must be `pathlib.Path` objects resolved relative to the config file's location. The module must pass `mypy --strict` and `ruff check`.

## Context
Task 00001 created the project scaffold including `config.yaml` at `/home/buddy/projects/corpus-council/config.yaml` and the package structure at `src/corpus_council/`. The `config.yaml` has this structure:
```yaml
# API keys are not stored here — use environment variables (ANTHROPIC_API_KEY, etc.)
llm:
  provider: anthropic
  model: claude-3-5-haiku-20241022
embedding:
  provider: sentence-transformers
  model: all-MiniLM-L6-v2
data_dir: data
corpus_dir: corpus
council_dir: council
templates_dir: templates
plans_dir: plans
chunking:
  max_size: 512
retrieval:
  top_k: 5
```

The `AppConfig` dataclass must expose these fields:
- `llm_provider: str`
- `llm_model: str`
- `embedding_provider: str`
- `embedding_model: str`
- `data_dir: Path`
- `corpus_dir: Path`
- `council_dir: Path`
- `templates_dir: Path`
- `plans_dir: Path`
- `chunk_max_size: int`
- `retrieval_top_k: int`

All `Path` fields must be resolved: if the value in `config.yaml` is a relative path, resolve it relative to the directory containing `config.yaml` (not the cwd). Use `Path(config_path).parent / value` then `.resolve()`.

The function must raise `FileNotFoundError` with a clear message if the config file does not exist, and `KeyError` or `ValueError` with a clear message if a required key is missing.

Use `from __future__ import annotations` at the top of the file.

No API keys are read from config — they come from environment variables elsewhere.

## Steps
1. Create `src/corpus_council/core/config.py` with `from __future__ import annotations`.
2. Define a `@dataclass` named `AppConfig` with all fields listed in Context, fully type-annotated.
3. Implement `load_config(path: str | Path) -> AppConfig` that:
   a. Checks the path exists; raises `FileNotFoundError` if not.
   b. Opens and parses the YAML with `yaml.safe_load`.
   c. Resolves all path values relative to the config file directory.
   d. Returns a populated `AppConfig` instance.
4. Export `AppConfig` and `load_config` from the module (they are the public API).
5. Run `uv run mypy src/corpus_council/core/config.py` and fix all errors.
6. Run `uv run ruff check src/corpus_council/core/config.py` and fix all errors.
