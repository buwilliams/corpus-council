# Task 00015: Implement Typer CLI with all five commands

## Role
api-designer

## Objective
Implement `src/corpus_council/cli/main.py` with a Typer app and five commands: `chat`, `collect`, `ingest`, `embed`, and `serve`. The CLI entry point `corpus-council` is already declared in `pyproject.toml` from Task 00001. All commands must call core modules directly. Must pass `ruff check` and `ruff format --check`.

**Command specifications (from `api-designer.md`):**

`corpus-council chat <user_id>`:
- Validate `user_id` with `validate_id`.
- Interactive REPL: print a welcome message, then loop reading from `sys.stdin`.
- Each non-empty line: call `run_conversation(user_id, message, config, store, llm)`.
- Print `> {response}` to stdout.
- Exit gracefully on EOF (`Ctrl+D`) or when user types `quit` or `exit`.

`corpus-council collect <user_id> [--session <session_id>]`:
- Validate `user_id` with `validate_id`.
- If `--session` provided: validate `session_id` with `validate_id`.
- If no `--session`: generate `session_id = str(uuid.uuid4())` and call `start_collection` (need `--plan` option too — add `--plan <plan_id>` as required when no session given).
- Interactive loop: print prompt, read response, call `respond_collection`, print next prompt.
- Exit when `status == "complete"`.

`corpus-council ingest <path>`:
- Load config, create `IngestResult = ingest_corpus(config)` (using supplied path to override `config.corpus_dir` if different, or use config as-is for the default corpus dir).
- Print `Processed {files_processed} files, created {chunks_created} chunks.`

`corpus-council embed`:
- Load config, call `embed_corpus(config)`.
- Print `Created {vectors_created} vectors.`

`corpus-council serve [--host <host>] [--port <port>]`:
- Defaults: `host="127.0.0.1"`, `port=8000`.
- Launch uvicorn: `uvicorn.run("corpus_council.api.app:app", host=host, port=port)`.

Config loading: all commands load `config = load_config(Path("config.yaml"))` at the start. If the config file is not found, print an error and exit with code 1.

Use `from __future__ import annotations`. No inline prompt strings. Use `typer.Typer()` as the app.
