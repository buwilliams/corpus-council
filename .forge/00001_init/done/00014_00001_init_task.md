# Task 00014: Implement FastAPI app, routers, and exception handlers

## Role
api-designer

## Objective
Implement the FastAPI application at `src/corpus_council/api/app.py` and three router files: `src/corpus_council/api/routers/conversation.py`, `src/corpus_council/api/routers/collection.py`, and `src/corpus_council/api/routers/corpus.py`. All six endpoints must be functional, call the correct core module functions, use the Pydantic models from Task 00013, validate inputs with `validate_id` from Task 00009, and return correct HTTP status codes. Global exception handlers must sanitize error responses. Must pass `ruff check` and `ruff format --check`.

**App structure (`app.py`):**
- Create `FastAPI` instance with title `"Corpus Council"`.
- Load `AppConfig` from `config.yaml` (use `load_config(Path("config.yaml"))`).
- Create shared `FileStore(config.data_dir)` and `LLMClient(config)` as module-level singletons.
- Register routers with their prefix paths.
- Add exception handlers:
  - `FileNotFoundError` → 404, `{"error": "Resource not found"}`
  - `ValueError` → 422, `{"error": str(exc)}`
  - General `Exception` → 500, `{"error": "Internal server error"}` (log actual message to server log, not response body)

**Endpoint implementations:**

`POST /conversation` (routers/conversation.py):
- Validate `user_id` with `validate_id`.
- Call `run_conversation(user_id, message, config, store, llm)`.
- Return `ConversationResponse(response=result.response, user_id=user_id)`.

`POST /collection/start` (routers/collection.py):
- Validate `user_id` and `plan_id` with `validate_id`.
- Generate `session_id = str(uuid.uuid4())`.
- Call `start_collection(user_id, plan_id, session_id, config, store, llm)`.
- Return 201 with `CollectionStartResponse`.
- On `FileNotFoundError` (plan not found): return 404 `{"error": "plan not found"}`.

`POST /collection/respond` (routers/collection.py):
- Validate `user_id`, `session_id`.
- Call `respond_collection(user_id, session_id, message, config, store, llm)`.
- Return `CollectionRespondResponse`.
- On `FileNotFoundError` (session not found): return 404 `{"error": "session not found"}`.

`GET /collection/{user_id}/{session_id}` (routers/collection.py):
- Validate `user_id`, `session_id`.
- Call `get_collection_status(user_id, session_id, store)`.
- Return `CollectionStatusResponse`.
- On `FileNotFoundError`: return 404 `{"error": "session not found"}`.

`POST /corpus/ingest` (routers/corpus.py):
- Validate `path` is within corpus dir using `validate_path_containment(Path(request.path), config.corpus_dir, "corpus path")`.
- Call `ingest_corpus(config)` (override `config.corpus_dir` with the validated path if needed, or use as-is if it matches).
- Return `CorpusIngestResponse`.

`POST /corpus/embed` (routers/corpus.py):
- Call `embed_corpus(config)`.
- Return `CorpusEmbedResponse`.

Routers must be `APIRouter` instances included in app.py. Each router file imports the shared `config`, `store`, `llm` from `app.py` or via a dependency injection pattern (simple: pass config/store/llm as module-level imports from app).

Use `from __future__ import annotations`. No inline prompt strings.
