# Task 00020: Integration tests — all six FastAPI API endpoints

## Role
tester

## Objective
Write `tests/integration/test_api.py` with integration tests for all six FastAPI endpoints. Tests use `httpx.AsyncClient(app=app, base_url="http://test")` (in-process, no server needed). All tests use real files from `tmp_path` fixtures, real `FileStore`, and `TestLLMClient` that overrides HTTP transport only. Must pass with `uv run pytest`.

## Context
Task 00014: FastAPI app at `src/corpus_council/api/app.py` — the `app` object. The app loads config from `config.yaml` at startup; tests must override this so the app uses `test_config` pointing to `tmp_path`.
Task 00013: Pydantic models in `src/corpus_council/api/models.py`.
Task 00016: `test_config`, `file_store`, `corpus_dir`, `council_dir`, `plans_dir`, `templates_dir` fixtures in `tests/conftest.py`.
Task 00018: `TestLLMClient` pattern.

To override the app's config/store/llm for tests without mocking — use FastAPI's dependency override mechanism or monkeypatch the module-level singletons in `app.py` before the test. Use `pytest.mark.anyio` or `pytest-asyncio` for async tests.

The app module (`src/corpus_council/api/app.py`) exposes module-level `config`, `store`, and `llm` objects. In tests, use `monkeypatch` to replace these with test instances:
```python
import corpus_council.api.app as app_module
monkeypatch.setattr(app_module, "config", test_config)
monkeypatch.setattr(app_module, "store", FileStore(test_config.data_dir))
monkeypatch.setattr(app_module, "llm", TestLLMClient(test_config))
```

**Tests to write in `tests/integration/test_api.py`:**

- `test_post_conversation_returns_200` — POST `{"user_id": "user0001", "message": "Hello"}` → status 200, response has `response` and `user_id` fields
- `test_post_conversation_rejects_extra_fields` — POST with extra field `{"user_id": "u0001", "message": "hi", "foo": "bar"}` → status 422
- `test_post_collection_start_returns_201` — POST `{"user_id": "user0001", "plan_id": "signup"}` → status 201, response has `session_id`, `first_prompt`
- `test_post_collection_start_returns_404_for_missing_plan` — POST `{"user_id": "user0001", "plan_id": "nonexistent"}` → status 404, body has `error` field
- `test_post_collection_respond_returns_200` — start a session then POST respond → status 200, body has `status`, `prompt`, `collected`
- `test_post_collection_respond_returns_404_for_missing_session` — POST with nonexistent `session_id` → status 404
- `test_get_collection_status_returns_200` — start session then GET status → status 200, body has `user_id`, `session_id`, `status`, `collected`, `created_at`
- `test_get_collection_status_returns_404_for_missing_session` — GET with nonexistent session → status 404
- `test_post_corpus_ingest_returns_200` — POST `{"path": str(corpus_dir)}` → status 200, body has `chunks_created`, `files_processed`
- `test_post_corpus_embed_returns_200` — first ingest, then POST `{}` to `/corpus/embed` → status 200, body has `vectors_created`
- `test_post_conversation_invalid_user_id_returns_422` — POST with `user_id` containing path traversal chars → status 422

Use `pytest_asyncio.fixture` and `AsyncClient`. Mark async tests with `@pytest.mark.asyncio`.
