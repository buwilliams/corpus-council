# Task 00016: Create test fixtures — real corpus, council personas, collection plan, and conftest.py

## Role
tester

## Objective
Create `tests/conftest.py` with pytest fixtures that provide real test corpus files, council persona markdown files, a collection plan, and a configured `AppConfig` pointing to `tmp_path`. All fixtures write real files — no mocks. These fixtures will be shared by all unit and integration tests in Tasks 00017–00021.

## Context
Task 00001 created `tests/`, `tests/unit/`, and `tests/integration/` with `__init__.py` files. Task 00002 created `AppConfig`. Task 00003 created `FileStore`. Task 00006 established the council member markdown format (YAML front matter with `name`, `persona`, `primary_lens`, `position`, `role_type`, `escalation_rule` + body prose). Task 00012 established the collection plan format (YAML front matter with `fields` list).

**Fixtures to create in `tests/conftest.py`:**

1. `corpus_dir(tmp_path) -> Path` — writes at least 3 `.md` files and 1 `.txt` file with diverse content to `tmp_path / "corpus"`. Returns the directory path.

2. `council_dir(tmp_path) -> Path` — writes exactly 3 council member files to `tmp_path / "council"`:
   - `critic.md`: `name: Adversarial Critic`, `position: 3`, `role_type: critic`, escalation_rule that can be triggered
   - `analyst.md`: `name: Domain Analyst`, `position: 2`, `role_type: domain_specialist`
   - `synthesizer.md`: `name: Final Synthesizer`, `position: 1`, `role_type: synthesizer`

3. `plans_dir(tmp_path) -> Path` — writes `signup.md` plan with at least 2 required fields (`first_name`, `email`) to `tmp_path / "plans"`. Returns the directory path.

4. `templates_dir(tmp_path) -> Path` — copies or recreates all 6 templates from the project `templates/` directory into `tmp_path / "templates"`. This ensures tests use the real templates. Returns the directory path.

5. `data_dir(tmp_path) -> Path` — creates `tmp_path / "data"` directory. Returns path.

6. `test_config(tmp_path, corpus_dir, council_dir, plans_dir, templates_dir, data_dir) -> AppConfig` — assembles a real `AppConfig` pointing all paths at `tmp_path`-based directories.

7. `file_store(test_config) -> FileStore` — returns `FileStore(test_config.data_dir)`.

Each fixture uses the `tmp_path` pytest built-in, which is automatically isolated per test. Never write to the real `data/`, `corpus/`, or `council/` directories.

Council member file format (see Task 00006):
```markdown
---
name: Adversarial Critic
persona: A sharp, skeptical critic who challenges every claim
primary_lens: factual accuracy
position: 3
role_type: critic
escalation_rule: Halt if response contains factually false claims
---
I challenge every assertion with evidence-based scrutiny.
```

Collection plan format (see Task 00012):
```markdown
---
title: User Signup
fields:
  - name: first_name
    description: The user's first name
    required: true
    validation_rule: Must be a non-empty alphabetic string
  - name: email
    description: The user's email address
    required: true
    validation_rule: Must be a valid email format
---
Collect user signup information.
```
