# Task 00005: Implement embeddings.py and retrieval.py — ChromaDB embedding pipeline and semantic search

## Role
data-engineer

## Objective
Implement `src/corpus_council/core/embeddings.py` with `embed_corpus(config: AppConfig) -> EmbedResult` that generates sentence-transformer embeddings for all corpus chunks (loaded from `data/chunks/`) and upserts them into a local ChromaDB collection. Implement `src/corpus_council/core/retrieval.py` with `retrieve_chunks(query: str, config: AppConfig, top_k: int | None = None) -> list[ChunkResult]` that embeds the query and returns the top-K most similar chunks. Both modules must pass `mypy --strict` and `ruff check`.

## Context
Task 00002 created `AppConfig`. Task 00004 created corpus chunks as JSON files at `data/chunks/{source_hash}/{chunk_index}.json` with schema:
```json
{
  "chunk_id": "uuid4-string",
  "source_file": "relative/path/to/file.md",
  "source_hash": "sha256-hex",
  "chunk_index": 0,
  "text": "chunk content",
  "char_start": 0,
  "char_end": 512
}
```

ChromaDB must use `chromadb.PersistentClient(path=str(config.data_dir / "embeddings"))`. Collection name comes from config (add `chroma_collection: str` with default `"corpus"` to `AppConfig` and `config.yaml`, or use a hardcoded but config-driven name — the collection name must be read from `config.yaml`).

Embedding provider selection:
- If `config.embedding_provider == "sentence-transformers"`: use `sentence_transformers.SentenceTransformer(config.embedding_model)` to encode chunk texts. Pass the embedding as a list of floats to ChromaDB.
- Other providers may be added later; raise `ValueError(f"Unknown embedding provider: {config.embedding_provider}")` for unknown providers.

`embed_corpus` must:
1. Load all chunk JSON files from `config.data_dir / "chunks"` (glob `**/*.json`).
2. For each chunk, check if `chunk_id` is already in ChromaDB using `collection.get(ids=[chunk_id])`. If present (and `upsert` semantics preferred), use `collection.upsert()` to update.
3. Embed all chunk texts in batch using the configured provider.
4. Upsert embeddings to ChromaDB with: `ids=[chunk_id]`, `embeddings=[vector]`, `documents=[text]`, `metadatas=[{"source_file": ..., "chunk_index": ...}]`.
5. Return `EmbedResult(vectors_created=int)`.

`EmbedResult` is a `dataclass`:
```python
@dataclass
class EmbedResult:
    vectors_created: int
```

`retrieve_chunks` must:
1. Embed the `query` text using the same provider.
2. Query ChromaDB collection with `n_results=top_k` (use `config.retrieval_top_k` if `top_k` is None).
3. Return `list[ChunkResult]` where each `ChunkResult` is:
```python
@dataclass
class ChunkResult:
    chunk_id: str
    text: str
    source_file: str
    chunk_index: int
    distance: float
```

The ChromaDB collection name must be read from `config.yaml`. Add `chroma_collection: str = "corpus"` to `AppConfig` (update `config.py` and `config.yaml` from Task 00002/00001 accordingly — add the field with a default of `"corpus"`).

Use `from __future__ import annotations`.

## Steps

   a. Import `chromadb`, embedding utilities, `AppConfig`, `ChunkResult`.

   c. Implement `retrieve_chunks(query: str, config: AppConfig, top_k: int | None = None) -> list[ChunkResult]`.
